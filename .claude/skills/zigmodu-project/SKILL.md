# ZigModu Project Navigation

## Project Recognition

A ZigModu project is identified by:
- `build.zig.zon` with `zigmodu` dependency
- `src/main.zig` importing `zigmodu`
- `src/modules/` directory with subdirectories per module (flat or subsystem-nested)
- `src/shared/` shared kernel

## Quick Orientation

```bash
grep zigmodu build.zig.zon    # framework version
ls src/modules/                # module list (may include subsystem dirs)
/usr/bin/find src/modules -name "module.zig" | wc -l  # module count
ls src/shared/                 # shared kernel
```

## Project Structure

```
src/
├── main.zig              # ⛔ @generated — entry, DB config, module wiring
├── shared/               # shared kernel (AI-editable)
├── business/             # cross-module orchestration (AI)
├── compat/               # legacy compatibility (migration)
└── modules/
    ├── <name>/            # flat module (no subsystem prefix)
    │   ├── module.zig     # ⛔ lifecycle + barrel re-exports + health
    │   ├── model.zig      # ⛔ data structs + json_names
    │   ├── persistence.zig # ⛔ ORM Repository(T)
    │   ├── service.zig    # ⛔ CRUD + validate
    │   ├── api.zig        # ⛔ REST handlers
    │   └── ext/           # ✅ AI writes here
    └── <subsystem>/       # auto-detected (shop_, crm_, etc.)
        └── <module>/
            └── (same 5 files + ext/)
```

### Subsystem Detection (auto)

Tables with shared first-segment prefix (`shop_orders`, `shop_products`) auto-grouped into `shop/` subsystem. 2+ modules sharing prefix triggers subsystem creation. Single-module prefixes stay flat.

## Import Conventions

```zig
// Flat module
const users = @import("modules/users/module.zig");

// Subsystem module (var name uses _, path uses /)
const shop_orders = @import("modules/shop/orders/module.zig");
const crm_customers = @import("modules/crm/customers/module.zig");

// Domain imports (within modules)
const data = @import("zigmodu").data;      // persistence.zig
const http = @import("zigmodu").http;      // api.zig
const zigmodu = @import("zigmodu");        // service.zig, module.zig
```

## ⛔ HARD BLOCK: Generated Files

Files marked `//! ⛔ @generated by zmodu — DO NOT EDIT` are owned by `zmodu scaffold`. **Never edit.** Changes: update schema.sql → `zmodu scaffold --force`.

## Scaffold Commands

```bash
zmodu scaffold --sql schema.sql --name <name>               # from SQL
zmodu scaffold --from-db "postgresql://user@host/db" --name <name>  # from live DB
zmodu scaffold --from-db /path/to/db.sqlite --name <name>   # from SQLite
```

## Key Types Reference

| Type | Import Path | Usage |
|------|------------|-------|
| `Context` | `zigmodu.http.Context` | HTTP request/response |
| `SqlxBackend` | `zigmodu.data.SqlxBackend` | DB backend |
| `Repository(T)` | `zigmodu.data.Repository(T)` | Typed ORM repo |
| `PageResult(T)` | `zigmodu.data.orm.PageResult(T)` | Paginated results |
| `Application` | `zigmodu.Application` | App lifecycle |
