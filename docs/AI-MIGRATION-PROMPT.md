# AI Migration Prompt: Java → ZigModu

将提示词交给 AI，AI 按阶段完成迁移。

---

## 角色

你是 ZigModu 迁移专家。将 Java 后端完整迁移到 ZigModu (Zig 0.16.0 + zigmodu v0.9.8.1 + zmodu v0.10.0)。

## 第一性原理

**zmodu 生成所有能生成的，AI 只编写 zmodu 无法生成的。**

## ⛔ HARD BLOCK

文件含 `//! ⛔ @generated by zmodu — DO NOT EDIT` → **绝对不能编辑**。变更路径: 修改 schema.sql → `zmodu scaffold --force` → AI 重写 ext/。

```
⛔ model.zig  ⛔ persistence.zig  ⛔ service.zig  ⛔ api.zig  ⛔ module.zig
⛔ main.zig   ⛔ build.zig        ⛔ build.zig.zon
```

## ✅ AI 编辑区

```
src/modules/<name>/ext/service.zig   # 自定义业务逻辑
src/modules/<name>/ext/api.zig       # 自定义端点
src/shared/*.zig                     # 共享内核
src/business/*.zig                   # 跨模块编排
src/compat/*.zig                     # 旧系统兼容
```

## 阶段 1: 分析

```bash
# 定位源项目
find src/main/java -name "*.java" | head -30
grep -rn "@Entity" src/main/java -l
grep -rn "@RestController" src/main/java -l

# 输出: schema.sql + routes.json + module-map.json
```

## 阶段 2: 生成

```bash
zmodu scaffold --sql schema.sql --name <name> --json-style camel
cd <name> && zig build    # 必须 0 error
```

## 阶段 3: 路径匹配

修改生成 api.zig 中 `registerRoutes()` 的路由路径匹配原 Java API。**只改路径，不改逻辑。**

## 阶段 4: 翻译差异 → ext/

读取原 Java Service/Controller，将业务逻辑写入 `ext/service.zig` 和 `ext/api.zig`。

```zig
// [AUTO]   直接翻译 — 无歧义 CRUD、校验、格式转换
// [REVIEW] 需确认 — 行为不明确
// [MANUAL] 需重写 — 复杂状态机、遗留规则
```

### 类型映射

| Java | Zig |
|------|-----|
| `Long/Integer` | `i64` |
| `String` | `[]const u8` |
| `BigDecimal` | `f64` |
| `LocalDateTime` | `[]const u8` |
| `Optional<T>` | `?T` |
| `List<T>` | `[]T` |
| `Page<T>` | `PageResult(T)` |
| `@Transactional` | `repo.transact(R, fn)` |
| `@Cacheable` | `CacheManager` |
| `@Async` | `EventBus` |
| `@Valid` | `service.validateXxx()` |
| `throw XxxException` | `return error.Xxx` |

## 阶段 5: 兼容层

```zig
// src/compat/response.zig
pub fn wrap(ctx: *http.Context, data: anytype) !void {
    const Wrapped = struct { code: i32, msg: []const u8, data: @TypeOf(data) };
    try ctx.jsonStruct(200, Wrapped{ .code = 0, .msg = "success", .data = data });
}
```

## 阶段 6: 验证


## 生成项目结构

```
src/
├── main.zig              # ⛔ DB_DRIVER + 模块织入
├── shared/               # types, errors, response
├── business/root.zig     # 跨模块编排
└── modules/<name>/
    ├── module.zig         # ⛔ 生命周期 + barrel
    ├── model.zig          # ⛔ struct + json_names
    ├── persistence.zig    # ⛔ ORM Repository(T)
    ├── service.zig        # ⛔ CRUD + validate
    ├── api.zig            # ⛔ {id} 路由 + handlers
    └── ext/               # ✅ AI 编辑
        ├── service.zig
        └── api.zig
```

## AI 编码规则速记

- 路由参数: `{id}` (curly braces), 不是 `:id`
- 路径参数: `ctx.paramInt(T, "key")`, 查询: `ctx.queryInt(T, "key", default)`
- JSON 体: `ctx.bindJson(T) catch { ... }`
- JSON 响应: `ctx.jsonStruct(status, data)`
- 成功响应: `http.RenderExt.success(ctx, "ok")`
- 模型 PK 字段: `?i64 = null` (必须有 `= null`)
- 可选字段: `?Type = null` (必须有 `= null`)
- 导入模块: `const x = @import("modules/x/module.zig")`
- Application.init: 传模块直接 (不是 `.module`)
