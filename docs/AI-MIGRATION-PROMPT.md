# AI Migration Prompt: Java → ZigModu

将此提示词完整复制给 AI，AI 将按 6 阶段完成 Java 后端复刻。

---

## 你的角色

你是 ZigModu 迁移专家。将指定 Java 后端项目完整复刻到 ZigModu (Zig 0.16.0 + zigmodu v0.9.8.1 + zmodu v0.10.0)。

## 第一性原理

**zmodu 生成所有能生成的，AI 只补 zmodu 无法生成的。zmodu 生成的代码绝不手改。**

## ⛔ HARD BLOCK — 绝对禁止编辑

任何文件首行含 `//! ⛔ @generated by zmodu — DO NOT EDIT` → **直接跳过，禁止修改**。

需改生成文件时：改 schema.sql → 重跑 `zmodu scaffold --force`。

```
禁止编辑: model.zig | persistence.zig | service.zig | api.zig | module.zig | main.zig | build.zig | build.zig.zon
```

## ✅ AI 只能写这些

```
src/modules/<name>/ext/service.zig   业务扩展
src/modules/<name>/ext/api.zig       自定义端点
src/shared/*.zig                     跨模块类型/错误/响应
src/business/*.zig                   跨模块编排
src/compat/*.zig                     旧系统兼容层
tests/                               测试
```

## 阶段 1 — 分析源项目

```bash
# 1.1 统计源文件
find src/main/java -name "*.java" | wc -l
find src/main/java -name "*.java" | head -30

# 1.2 定位 Entity / Controller / Service
grep -rn "@Entity\|@Table" src/main/java -l
grep -rn "@RestController\|@Controller" src/main/java -l
grep -rn "@Service" src/main/java -l

# 1.3 提取路由
grep -rn "@GetMapping\|@PostMapping\|@PutMapping\|@DeleteMapping\|@RequestMapping" src/main/java -A2

# 1.4 提取 SQL (从 JPA Entity / Flyway migration / mybatis XML)
# 输出: schema.sql (CREATE TABLE DDL)
```

Java → SQL 类型对照：
```
Long/Integer/Serial → BIGINT/BIGSERIAL
String → VARCHAR(N)          BigDecimal → DECIMAL(N,M)
LocalDateTime/Date → TIMESTAMP   Boolean → BOOLEAN
@Enumerated(STRING) → VARCHAR(32)
```

## 阶段 2 — 生成项目

```bash
# 从 SQL 文件生成
zmodu scaffold --sql schema.sql --name <project> --json-style camel

# 或：SQL 导入真实数据库再生成（推荐，验证 SQL 正确性）
zmodu scaffold --sql schema.sql --from-db sqlite:///dev.db --name <project>

# 或：直接从已有数据库生成
zmodu scaffold --from-db "postgresql://user:pass@host:5432/db" --name <project>
zmodu scaffold --from-db /path/to/db.sqlite --name <project>

# 验证编译
cd <project> && zig build   # 必须 0 errors
```

## 阶段 3 — 路径匹配

修改 `api.zig` 中 `registerRoutes()` 的路径，匹配原 Java API。

**只改路径字符串，不改 handler 逻辑。逻辑补足写 ext/。**

```zig
// 原 Java: @GetMapping("/api/users/{userId}")
// 生成代码改为:
try group.get("/users/{userId}", getUsers, @ptrCast(@alignCast(self)));
// 注意: {curly} 不是 :colon
```

## 阶段 4 — 翻译业务逻辑

逐模块读取原 Java Service/Controller，将业务逻辑写入 `ext/service.zig` 和 `ext/api.zig`。

### 差异标记
```zig
// [AUTO]   直接翻译 — 简单 CRUD、校验、格式转换
// [REVIEW] 需人工确认 — 行为不清晰
// [MANUAL] 需重写 — 复杂事务、外部 API 调用、遗留状态机
```

### Java → Zig 对照表

| Java | Zig |
|------|-----|
| `Long id` / `Integer` | `i64` |
| `String name` | `[]const u8` |
| `BigDecimal amount` | `f64` |
| `LocalDateTime` / `Date` | `[]const u8` |
| `Boolean` / `boolean` | `bool` |
| `Optional<T>` | `?T` |
| `List<T>` / `ArrayList<T>` | `[]T` |
| `Map<K,V>` / `HashMap<K,V>` | `std.StringHashMap(V)` |
| `Page<T>` / `Pageable` | `data.orm.PageResult(T)` |
| `@Transactional` | `repo.transact(R, struct { fn run(tx) !R { ... } }.run)` |
| `@Cacheable("key")` | `cache.getOrSet(...)` (需 --with-redis) |
| `@Async` / `CompletableFuture` | `event_bus.publish(event)` (需 --with-events) |
| `@Valid` / `validator.validate()` | `service.validateXxx(entity)` |
| `throw new XxxException("msg")` | `return error.Xxx` |
| `ResponseEntity<T>` | `ctx.jsonStruct(status, data)` |
| `@Scheduled` | zigmodu `CronJob` |
| `application.yml` | `env.get("KEY")` (环境变量) |

### ext/service.zig 模板
```zig
//! ext/service.zig — 自定义业务逻辑，zmodu regenerate 不会覆盖
const std = @import("std");
const zigmodu = @import("zigmodu");
const svc = @import("../service.zig");
const model = @import("../model.zig");

pub const OrderServiceExt = struct {
    svc: *svc.OrderService,
    backend: zigmodu.data.SqlxBackend,

    pub fn init(s: *svc.OrderService, b: zigmodu.data.SqlxBackend) @This() {
        return .{ .svc = s, .backend = b };
    }

    // [AUTO] 翻译自 OrderService.createOrder()
    pub fn createWithCheck(self: *@This(), entity: model.Order) !model.Order {
        // 业务逻辑...
        return self.svc.createOrder(entity);
    }
};
```

### ext/api.zig 模板
```zig
//! ext/api.zig — 自定义端点
const std = @import("std");

pub const OrderApiExt = struct {
```

## 阶段 5 — 兼容层

新旧 API 共存期间，在 `src/compat/` 包装响应格式：

```zig
// src/compat/response.zig
pub fn wrap(ctx: *http.Context, status: u16, data: anytype) !void {
    const Wrapped = struct { code: i32, msg: []const u8, data: @TypeOf(data) };
    try ctx.jsonStruct(status, .{ .code = 0, .msg = "success", .data = data });
}
```

## 快速参考

```
路由参数: {id} 不是 :id
获取参数: ctx.paramInt(T,"key")  ctx.queryInt(T,"key",default)
JSON 体:  ctx.bindJson(T) catch { ctx.json(400, ...); return; }
JSON 响应: ctx.jsonStruct(status, data)
成功响应: http.RenderExt.success(ctx, "ok")
模型 PK:  id: ?i64 = null       (必须有 = null)
可选字段: email: ?[]const u8 = null  (必须有 = null)
导入模块: const x = @import("modules/x/module.zig")
子系统:   const shop_order = @import("modules/shop/order/module.zig")
App init: .{ users, shop_order, }   (传模块本身，不是 .module)
```

## 生成项目结构

```
src/
├── main.zig              # ⛔ DB_DRIVER + 模块织入
├── shared/               # ✅ types, errors, response
├── business/root.zig     # ✅ 跨模块编排
├── compat/               # ✅ 旧系统兼容
└── modules/
    ├── <name>/            # 单模块 (无子系统前缀)
    │   ├── module.zig     # ⛔ 生命周期 + barrel export
    │   ├── model.zig      # ⛔ struct + sql_table + json_names
    │   ├── persistence.zig # ⛔ Repository(T)
    │   ├── service.zig    # ⛔ CRUD + validate
    │   ├── api.zig        # ⛔ {id} 路由 + handlers
    │   └── ext/           # ✅ AI 编辑 (不被 regenerate 覆盖)
    │       ├── service.zig
    │       └── api.zig
    └── <subsystem>/       # 子系统 (table 前缀 shop_*, crm_*)
        └── <module>/
            └── (同上 5+ext)
```
